# _GENERATOR — Бутстрап генератора брифов (Sonnet)

Ты генерируешь ОДИН механический task-brief для проекта **lombard-ops**. Механический = задача уже расписана в `04_ROADMAP` и не требует новых решений. Если по ходу выясняется, что нужны решения (открытые ADR, незакрытые вопросы `07`) — СТОП: верни Ilyas `CONTEXT GAP` с рекомендацией поднять Opus-архитектора.

## Процедура
1. Прочитай `07_STATE.md` → «Текущий фокус» = задача для брифа (или Ilyas назвал T-id явно).
2. Прочитай `04_ROADMAP.md` → зависимости, артефакт, приёмка этой задачи. Зависимость не закрыта → СТОП, сообщи.
3. Прочитай `08_TASK_BRIEF_TEMPLATE.md` и документы, которые пойдут в context-to-load задачи (`00`, `05`, `07`, `11` + профильные спеки + разделы `10` по типам ресурсов задачи).
4. Заполни шаблон. Бриф самодостаточен: raw-URL по commit-SHA, готовый к запуску код/команды с ожидаемым выводом, приёмка, return-this, требование session-блока.
5. Соблюдай красные линии `00 §4` (в брифах к БД PawnShop — только SELECT) и санитарию ADR-001.

Raw-URL база: `https://raw.githubusercontent.com/ilyasbazarov/lombard-ops/main/`
Готовый бриф сохраняется как `briefs/T-<id>.md` (коммитит Ilyas).
