# BRIEF · T-0-2 · Снятие полной структуры БД PawnShop

## Цель
Получить полную структуру БД PawnShop (все пользовательские таблицы, их русские описания, колонки с типами) в виде текстового файла для проектирования маппинга (T-0-3). Попутно: найти путь к файлу БД (.fdb) и рабочие креды подключения — они станут фактами `11_INFRA_FACTS` (сам пароль — вне репо).

## Входные артефакты (факты, уже установленные)
- Сервер: Windows Server, реальный адрес `192.168.99.100`; вход по RDP через `192.168.88.156` (проброс 3389).
- Firebird 2.5.9, служба `FirebirdServerPawnShop-3057`, порт 3057, слушает; путь `C:\Program Files (x86)\Firebird-2.5-PawnShop\`.
- Кодировка данных UTF-8. Админ-креды Windows — у Ilyas.
- Пути к .fdb и пароля SYSDBA НЕТ — их находит эта задача. Не угадывать (кроме санкционированной пробы дефолта в Шаге 3).

## Ограничения
- К БД — только чтение (00 §4). Все SQL этой задачи читают только системные RDB$-таблицы.
- Ничего не устанавливать на сервер, никаких сторонних утилит (IBExpert не нужен — хватит штатного isql).
- **Дамп структуры в публичный репо НЕ коммитить** — вендор считает структуру БД конфиденциальной. Файл передаётся архитектору в чат; в репо попадёт только маппинг используемых полей (`02 §3`). Зафиксировать это правилом в session-блоке (NEW_DECISIONS, accepted).
- Вывод терминала возвращать целиком, не пересказом (untrusted-правило 05 §I).

## Шаги (все — в RDP-сессии, PowerShell от администратора)

### Шаг 1 — найти файл БД
```powershell
Get-Content "C:\Program Files (x86)\Firebird-2.5-PawnShop\aliases.conf" -ErrorAction SilentlyContinue
Get-PSDrive -PSProvider FileSystem | Format-Table -Auto Name,Used,Free
```
Затем поиск .fdb по всем локальным дискам из вывода выше (пример для C:; повторить для D: и т.д., если есть):
```powershell
Get-ChildItem -Path C:\ -Filter *.fdb -Recurse -ErrorAction SilentlyContinue | Format-Table -Auto FullName,Length,LastWriteTime
```
Ожидание: aliases.conf покажет алиас → путь, либо поиск найдёт файл(ы). Рабочая база опознаётся по свежему LastWriteTime (сегодня) и большому размеру. Вывод — архитектору. Найдено несколько кандидатов → не выбирать самостоятельно, показать все.

### Шаг 2 — поискать сохранённые креды подключения PawnShop
Клиентское приложение PawnShop где-то хранит строку подключения. Найти его папку и просмотреть конфиги:
```powershell
Get-ChildItem "C:\Program Files (x86)" -Directory | Format-Table -Auto Name
Get-ChildItem "C:\Program Files" -Directory | Format-Table -Auto Name
```
В папке, похожей на PawnShop (не Firebird), найти конфиги с упоминанием порта/базы:
```powershell
Get-ChildItem "<ПАПКА_PAWNSHOP>" -Recurse -Include *.ini,*.cfg,*.xml,*.conf -ErrorAction SilentlyContinue |
  Select-String -Pattern "3057|\.fdb|password|user" -List | Format-Table -Auto Path,LineNumber,Line
```
Найденные строки подключения — архитектору (пароли не публиковать нигде дальше этого чата).

### Шаг 3 — проверить подключение isql
Санкционированная проба дефолта Firebird (SYSDBA/masterkey), затем креды из Шага 2, если дефолт не подошёл:
```powershell
& "C:\Program Files (x86)\Firebird-2.5-PawnShop\bin\isql.exe" -user SYSDBA -password masterkey "localhost/3057:<ПУТЬ_К_FDB>"
```
Внутри isql: `SHOW DATABASE;` затем `QUIT;`
- Успех: вывод параметров базы (ODS, страницы, charset).
- «Your user name and password are not defined» → дефолт не подошёл: взять креды из Шага 2; если и их нет → CONTEXT GAP, запрос пароля SYSDBA у админа/вендора. Подбор паролей запрещён.

### Шаг 4 — снять структуру
Создать SQL-скрипт (первая строка — комментарий, защита от BOM):
```powershell
New-Item -ItemType Directory -Path C:\temp -Force | Out-Null
@'
/* lombard-ops T-0-2 metadata dump */
SET LIST ON;
SELECT r.RDB$RELATION_NAME AS TABLE_NAME,
       CAST(SUBSTRING(r.RDB$DESCRIPTION FROM 1 FOR 300) AS VARCHAR(300)) AS TABLE_DESCR
FROM RDB$RELATIONS r
WHERE COALESCE(r.RDB$SYSTEM_FLAG,0)=0 AND r.RDB$VIEW_BLR IS NULL
ORDER BY 1;
SELECT rf.RDB$RELATION_NAME AS TABLE_NAME,
       rf.RDB$FIELD_POSITION AS POS,
       rf.RDB$FIELD_NAME AS COLUMN_NAME,
       f.RDB$FIELD_TYPE AS FTYPE,
       f.RDB$FIELD_SUB_TYPE AS FSUBTYPE,
       f.RDB$FIELD_LENGTH AS FLEN,
       f.RDB$FIELD_SCALE AS FSCALE,
       rf.RDB$NULL_FLAG AS NOTNULL,
       CAST(SUBSTRING(rf.RDB$DESCRIPTION FROM 1 FOR 300) AS VARCHAR(300)) AS COLUMN_DESCR
FROM RDB$RELATION_FIELDS rf
JOIN RDB$FIELDS f ON f.RDB$FIELD_NAME = rf.RDB$FIELD_SOURCE
JOIN RDB$RELATIONS r ON r.RDB$RELATION_NAME = rf.RDB$RELATION_NAME
WHERE COALESCE(r.RDB$SYSTEM_FLAG,0)=0 AND r.RDB$VIEW_BLR IS NULL
ORDER BY 1, 2;
'@ | Out-File -FilePath C:\temp\meta.sql -Encoding utf8
```
Выполнить (подставить пароль и путь из Шагов 1–3):
```powershell
& "C:\Program Files (x86)\Firebird-2.5-PawnShop\bin\isql.exe" -user SYSDBA -password <ПАРОЛЬ> -ch UTF8 -i C:\temp\meta.sql -o C:\temp\meta_out.txt "localhost/3057:<ПУТЬ_К_FDB>"
Get-Item C:\temp\meta_out.txt | Format-Table -Auto FullName,Length
Get-Content C:\temp\meta_out.txt -TotalCount 40
```
Ожидание: файл ненулевого размера; в первых строках — имена таблиц, русские описания читаемы (не кракозябры). Кракозябры → не пересоздавать ничего, показать вывод архитектору (вопрос кодировки решается флагом, не правкой БД).

### Шаг 5 — передать файл
Вариант А (обычно хватает): открыть `C:\temp\meta_out.txt` в Notepad → Ctrl+A → Ctrl+C → на Маке вставить в файл и загрузить в чат архитектору (буфер Windows App синхронизируется с Маком).
Вариант Б (если файл огромный и буфер давится): в Windows App → настройки подключения к PC → вкладка Folders → добавить папку Мака → скопировать файл в неё через Проводник сервера.
