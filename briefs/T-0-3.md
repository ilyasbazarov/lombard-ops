## Ограничения

- **К БД PawnShop — только SELECT** (`00 §4`, `05 §I`). Ни одной операции записи. Read-only пользователь/GRANT — это T-0-5, не здесь.
- **`SELECT *` запрещён**; только явные списки колонок; чтение системных `RDB$`-таблиц разрешено (`05 §II`, ADR-004).
- **Схему не угадывать**: любое `ТАБЛИЦА.колонка` — из дампа или живого запроса, иначе `CONTEXT GAP`.
- **Санитария (ADR-001 / ADR-009)**: в выводах и артефактах — никаких ПДн клиентов (имена/телефоны/номера договоров обезличивать), дамп не коммитится, в репо уходит только производный маппинг (`02 §3` + `mapping.json`); синтетические примеры — без специфичных колонок вендора.
- **Модель исполнения** (`05 §I`): Sonnet пишет SQL + ожидаемый вывод; запускает Ilyas и возвращает лог; done — только после подтверждённого человеком лога. Прод-код (коннектор) здесь не пишется — только документация маппинга и драфт `mapping.json`.
- **Untrusted-вывод** (`05 §I`): противоречивый/побитый RDP/Cloud Shell-вывод недостоверен — не примирять, запросить чистый повторный лог.

## Шаги

1. **Принять дамп.** Ilyas вставляет дамп структуры (T-0-2). Проверить, что есть: имена таблиц, колонки, типы, русские `description`. Чего-то нет → `CONTEXT GAP`.

2. **Снять/сверить эталон метаданных (RDB$).** Дать Ilyas на запуск read-only enumeration (Firebird 2.5, ERP-сервер). Это и переснимок для сверки с дампом, и seed отпечатка схемы для guard'а (ADR-004). Ожидаемый вывод — плоская таблица `tbl | col | pos | type | subtype | len | scale | notnull | descr`, ~1018 строк.

   ```sql
   -- meta_dump.sql — только чтение RDB$, без SELECT *; пользовательские таблицы (не системные, не вьюхи)
   SET LIST OFF;
   SELECT
     TRIM(rf.RDB$RELATION_NAME)  AS tbl,
     TRIM(rf.RDB$FIELD_NAME)     AS col,
     rf.RDB$FIELD_POSITION       AS pos,
     f.RDB$FIELD_TYPE            AS ftype,
     f.RDB$FIELD_SUB_TYPE        AS subtype,
     f.RDB$FIELD_LENGTH          AS flen,
     f.RDB$FIELD_SCALE           AS fscale,
     rf.RDB$NULL_FLAG            AS notnull,
     rf.RDB$DESCRIPTION          AS descr
   FROM RDB$RELATION_FIELDS rf
   JOIN RDB$FIELDS    f ON f.RDB$FIELD_NAME    = rf.RDB$FIELD_SOURCE
   JOIN RDB$RELATIONS r ON r.RDB$RELATION_NAME = rf.RDB$RELATION_NAME
   WHERE r.RDB$VIEW_BLR IS NULL
     AND (r.RDB$SYSTEM_FLAG IS NULL OR r.RDB$SYSTEM_FLAG = 0)
   ORDER BY rf.RDB$RELATION_NAME, rf.RDB$FIELD_POSITION;
   ```

   Запуск на ERP-сервере (cmd/PowerShell; пароль SYSDBA — из кред Ilyas, в бриф НЕ пишется):

   ```
   & "C:\Program Files (x86)\Firebird-2.5-PawnShop\bin\isql.exe" ^
       -user SYSDBA -password <из_кред_Ilyas_вне_репо> -ch UTF8 ^
       "localhost/3057:<путь_к_.fdb_из_T-0-2>" ^
       -i meta_dump.sql -o meta_dump.txt
   ```

3. **Шорт-лист кандидатов по дампу.** Для каждого canonical-поля `02 §3` (+ `02 §2`) выписать кандидатов `ТАБЛИЦА.колонка` с типом и русским `description`, обосновывающим выбор. Явно выделить критичные группы:
   - **Поля ADR-006** (для `balance_amount`): тело займа (principal/loan_amount), ставка (rate), правила пени. Если в БД есть только тело, а ставка/пени — глобальная настройка приложения вендора, а не поле договора → это НЕ маппинг, а расхождение с посылкой ADR-006: пометить `CONTEXT GAP` и вынести на Opus-архитектора (не «дотягивать»).
   - **Renewals** (`03 §2`): поле «Примечание» со связью на старый договор + маркеры операций «Погашение»/«Переоткрытие». Здесь только **локализовать поля**; детектор — это T-0-4, не T-0-3.
   - **VIN → vehicle_id**, issue_date/due_date, status_raw, госномер (для суррогата `VEH-{YYYYMMDD}-{госномер}` при отсутствии VIN).

4. **Подтверждающие SELECT'ы (после шорт-листа).** Для каждого кандидата — точечный `SELECT` с явными колонками, `FIRST n` (Firebird 2.5), без ПДн в выводе (клиентские идентификаторы исключить/обезличить), чтобы подтвердить семантику и реальные значения. Инстанцировать реальными именами из шага 3 (имена — из дампа/enumeration, не из головы). Пример-шаблон:

   ```sql
   -- confirm_<canonical>.sql — заменить <TBL>/<col*> реальными из шага 3; без SELECT *, без ПДн
   SELECT FIRST 20 <col_contract_id>, <col_principal>, <col_rate>, <col_issue_date>, <col_due_date>
   FROM <TBL_loans>
   WHERE <col_status_active_predicate>;   -- если предикат ещё не ясен — снять без WHERE, FIRST 20
   ```

5. **Свести логи → маппинг.** По подтверждённым человеком логам заполнить `02 §3` (canonical → Firebird `таблица.колонка`, тип, преобразование, примечание). Неразрешённые поля — оставить как явный `CONTEXT GAP` (не `⏳`-заглушку с догадкой). Собрать `mapping.json` (драфт): секция маппинга полей + секция отпечатка (подмножество used-таблиц с колонками/типами) для guard'а ADR-004.
