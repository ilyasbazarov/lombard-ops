# 08 · TASK_BRIEF_TEMPLATE — Шаблон брифа

**Статус:** STABLE. Каждый бриф — самодостаточен: свежий Sonnet-чат выполняет задачу, не зная о существовании других чатов.

```md
# BRIEF · T-<id> · <название задачи>

## Цель
<одним абзацем: что должно стать правдой после этой задачи>

## Context-to-load (обязательно, raw-URL по commit-SHA)
- 00_CHARTER: <url>
- 05_CONVENTIONS: <url>
- 07_STATE: <url>
- 11_INFRA_FACTS: <url>
- <спеки/разделы плейбука, относящиеся к задаче>

## Входные артефакты
<файлы, логи, значения — что дано. Чего-то нет → CONTEXT GAP, не угадывать>

## Ограничения
- К БД PawnShop — только SELECT (00 §4).
- <прочие для задачи>

## Шаги
1. <шаг: что написать, что человек запускает, ожидаемый вывод>
2. ...

## Приёмка (критерии done)
- [ ] <проверяемый критерий с ожидаемым логом/выводом>
- [ ] done только после подтверждённого человеком лога

## Return-this (что вернуть человеку)
- <артефакты: файлы для коммита, значения для 11, логи>
- Session-блок по формату 05 Часть III — обязательно
```

Правила: бриф не ссылается на «предыдущий чат»; всё нужное — внутри или в context-to-load. Код в брифе — готовый к copy-paste (heredoc для Cloud Shell). Для задач на ERP-сервере — команды под PowerShell/cmd Windows.
