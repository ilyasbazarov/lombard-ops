# 00 · CHARTER — Конституция проекта

**Версия:** 1.0 · **Статус:** STABLE (правки только через ADR) · **обновлено:** 2026-07-10

## 1. Миссия

Построить операционную систему автоломбарда «Омега Экспресс», закрывающую три диагностированных разрыва (аудит Блоков A–G, июнь 2026):

1. **Фильтр на входе** — неликвидный залог принимается без контроля владельца (RF-4: 31% неликвида; RF-1: оценка субъективна).
2. **Светофор по циклу** — статус 82 займов живёт в голове менеджера (RF-5 bus factor); реализация стартует поздно.
3. **Stop-Loss** — «ловушка идеальной цены» при реализации: ~$4,500/мес убытков от затяжных продаж.

Система реализуется в одну итерацию на целевом стеке — без промежуточных «версий для старта».

## 3. Шесть принципов архитектуры (наследуются из ТЗ, неизменны)

1. **Статус вычисляется, не хранится.** Производный статус не расходится с данными; историчность — журнал событий, не снапшоты.
2. **Один источник правды — журнал событий.** Таблица `events` в BigQuery, append-only.
3. **Слой канонизации.** Никакая функция не читает сырые данные PawnShop напрямую; дрейф формата чинится в одном месте.
4. **Все пороги — в конфиге.** Изменение бизнес-правила = правка конфига, не код и не деплой.
5. **VIN как vehicle_id.** Стабильный идентификатор объекта независимо от договоров и перерегистраций.
6. **Telegram — единственный интерфейс Исы.** Всё push-ом; кнопки — только при предложении ниже floor.

## 4. Красные линии (нарушать запрещено)

- **К БД PawnShop — только чтение.** Никогда ни одной операции записи, ни при каких обстоятельствах. Прод работает под read-only пользователем БД.
- **Все GCP-ресурсы — на аккаунте клиента.** Ilyas — editor, не owner.
- **Секреты и endpoint'ы контура клиента — вне репо** (репо публичный; полный список — ADR-001).
- **Веб-приложение — только за авторизацией** (Identity Platform, email+пароль, роли owner/manager/assessor). Анонимного доступа к данным ломбарда нет.
- **Правки STABLE/SEMI-STABLE документов — только через ADR** в `06_DECISIONS_LOG`.

## 5. Границы scope v1

- Один филиал, ~82 активных займа, три пользователя.
- Не строим: мобильные приложения, ML-прогнозы, интеграцию записи в PawnShop, бухгалтерию.
- Триггеры пересмотра архитектуры — `01_ARCHITECTURE §6`.

## 6. Критерии успеха

1. Эльхан проводит 100% осмотров через форму; 🔴-марки блокируются автоматически.
2. Иса получает утреннюю сводку и алерты по всем переходам статусов; ни один займ не «живёт в голове».
3. Все займы в реализации имеют три цены; решения ниже floor фиксируются с аудит-трейлом.
4. Система переживает обновление PawnShop без тихой порчи данных (guard схемы срабатывает стопом, не искажением).
