# 03 · BUSINESS_SPEC — Бизнес-логика

**Версия:** 1.0 · **Статус:** SEMI-STABLE · **обновлено:** 2026-07-10
Источник: Section7_v3_Final (архитектурное ТЗ, июнь 2026). Числовые пороги ниже — дефолты ТЗ; боевые значения живут в `config.json` и согласуются с Исой (T-0-9).

## 1. Статусная модель (вычисляется, не хранится)

День цикла = today − issue_date, с учётом renewals (продление сбрасывает отсчёт от renewed_date).

| День | Статус | Кто уведомляется | Действие |
|---|---|---|---|
| 0–22 | 🟢 Активный | — | тишина |
| 23–30 | 🟡 Внимание | Бектур | «Позвони клиенту, договор истекает через X дней» |
| 31 | 🟠 Просрочен | Бектур + Иса | информационно |
| 45 | 🟠 Пре-маркетинг | Эльхан + Иса | Эльхану чеклист; Исе информационно |
| 61 | 🔴 Реализация | Эльхан + Иса | протокол реализации + три цены |
| 61+7 / +14 / +30 | 🔴 | Иса | нет предложений / рекомендация −5% / сводка — информационно |
| любой | 🔴 особый | Иса | ⚠️ предложение НИЖЕ floor — единственный случай кнопок |

Валидация модели на реальных 82 займах с Бектуром до кода — T-0-6 (расхождения → корректировка логики, не данных).

## 2. Renewals

44% объектов перезакладываются — это норма, не edge case. Механизм в БД PawnShop двоякий («Погашение» / «Переоткрытие», связь в «Примечании») — детектор проектируется после T-0-4 по живым данным. Инвариант: VIN сохраняется, vehicle_id наследуется, счётчик цикла сбрасывается.

## 3. Справочник ликвидности (`vehicle_catalog`)

- Классы: 🟢 высокая / 🟡 средняя / 🔴 стоп (не принимается — форма блокирует сабмит).
- LTV считается от **скупочной** цены (сколько перекуп даёт сегодня), не от розницы Mashina.kg.
- Первичное заполнение: топ-30 марок из данных Блока A, калибровка с Эльханом и Исой. Обновление ~раз в квартал через /catalog.
- Дефолты ТЗ: Camry 70 🟢 LTV 75%; Prado/RX 🟢 70%; Mercedes E 🟡 55%; Range Rover / AMG / спорткары — 🔴.

## 4. Три цены (Stop-Loss)

| Цена | Формула | Назначение |
|---|---|---|
| Балансовая | тело + начисленные % + пени на сегодня (вычисляем сами, ADR-006) | P&L Исы; растёт каждый день — это правильно |
| Цена реализации | оценка на день 45 × (1 − коэф.износа × дни_с_дня_61 / 10) | таргет Эльхана; снижается — давление на скорость |
| Floor | тело × floor% (по классу марки из конфига) | минимум без одобрения; ниже — кнопки Исе |

Дефолты: floor 🟢 85%, 🟡 90%; износ −1% за 10 дней с дня 61; realization_price не ниже floor.
Backfill стартового портфеля: займы в 🔴 без оценки — Эльхан проходит форму осмотра, это «оценка на день дефолта» и обкатка формы.

## 5. Форма осмотра — дисконты

| Фактор | Шкала |
|---|---|
| Кузов | 0 / −3 / −7 / −15 / −20% |
| Толщиномер | <200: 0% · 200–400: −5% · >400: −15% |
| «Не на ходу» | стоп-фактор, форма блокируется |
| Пробег | <80K: 0% · 80–150K: −5% · >150K: −10% |
| Владельцы по ПТС | 1: 0% · 2: −3% · 3+: −7% |

`max_loan = base_price × (1 − Σ дисконтов) × ltv_max` — live-расчёт до сабмита. Фото: мин. 4, макс. 12 → GCS `photos/{VIN}/{timestamp}/`, типы inspection/parking/listing. После сабмита: запись в `assessments` + событие ASSESSMENT + информационное уведомление Исе (🟢/🟡; 🔴 не отправляется — заблокирован формой).

## 6. Психология управления (инварианты UX)

- Иса **не одобряет осмотры** — Справочник и есть его заранее зафиксированное решение.
- Все алерты Исе информационные, кроме «ниже floor» — там кнопки [Принять — зафиксировать убыток] / [Отклонить]; после нажатия бот редактирует сообщение с временем решения; нет редактирования = нет записи.
- Предложение выше floor — Эльхан решает сам, Иса только информируется.
- Утренняя сводка Исе в 09:05: счётчики по статусам, свежесть данных, события за вчера.
