# 09 · GLOSSARY — Термины и участники

**Статус:** STABLE

## Люди и стороны
- **Иса** — владелец ломбарда «Омега Экспресс». Роль `owner`.
- **Бектур** — операционный менеджер. Роль `manager`.
- **Эльхан** — оценщик и реализация залогов. Роль `assessor`.
- **Ilyas** — подрядчик, исполнитель проекта (человек в модели исполнения `05 §I`).
- **«Алгоритм»** — вендор ERP PawnShop.

## Домен
- **PawnShop** — ERP автоломбарда (Firebird 2.5.9). Источник данных, read-only.
- **Займ / договор** — выдача денег под залог авто; `contract_id`.
- **vehicle_id** — стабильный ID объекта = нормализованный **VIN**; суррогат `VEH-{YYYYMMDD}-{госномер}` при отсутствии VIN.
- **Renewal (продление)** — перезаклад; в PawnShop оформляется «Погашением» или «Переоткрытием договора». Сбрасывает счётчик цикла.
- **День цикла** — today − issue_date (с учётом renewal).
- **Светофор** — статусная модель 🟢/🟡/🟠/🔴 (`03 §1`).
- **Три цены** — балансовая / цена реализации / floor (`03 §4`).
- **Floor price** — минимальная цена продажи без одобрения Исы.
- **Скупочная цена** — сколько перекупщик платит сегодня наличными; база для LTV (не розница Mashina.kg).
- **Пре-маркетинг** — прогрев продажи с дня 45, до формальной реализации (день 61).
- **Backfill-осмотр** — осмотр займа, выданного до системы, для получения base_price.
- **Staleness** — устаревание данных конвейера (> порога из config.json).

## Процесс
- **ADR** — запись решения в `06`, append-only.
- **Session-блок** — структурированный вывод конца сессии (`05 §III`).
- **Task-brief** — самодостаточное задание для Sonnet-чата (`08`).
- **CONTEXT GAP** — стоп-сигнал anti-improvisation: не хватает входа, модель останавливается.
- **Tracer bullet** — тонкий сквозной срез конвейера e2e на живых данных (финиш Фазы 1).
- **Guard схемы** — сверка отпечатка метаданных Firebird с эталоном перед чтением (ADR-004).
- **INFRA_PATCH / STATE_PATCH** — патчи `11`/`07` из session-блока.
